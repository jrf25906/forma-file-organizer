## Brand Strategy

### Brand Positioning

**"Give your files form."**

Forma isn't just another file organizer. It's a beautifully designed tool for anyone who wants to get organized. While the app is crafted with the attention to detail that design-conscious users will appreciate, it's built for everyone who values a clean, well-organized digital life. Forma brings order to chaos through intelligent, refined organization that feels native to macOS.

---

### Target Audience

**Primary:**
- Anyone who wants to get organized and stay that way
- People who appreciate well-designed, thoughtful software
- Mac users looking for a modern file organization solution
- Those who have high standards for software quality and aesthetics

**Also Great For:**
- Design-conscious users who notice and appreciate craft
- Professionals managing multiple projects and file types
- Anyone who values form and function equally

**Not For:**
- Users who prefer maximum automation over control
- Enterprise IT departments (consumer-focused)

---

### Emotional Journey

Forma guides users through three emotional phases:

#### **Phase 1: SATISFYING** *(First Use)*
**The Feeling:** "This is exactly what I needed"

Users experience refined organization:
- Immediate visual clarity
- Appreciation for minimalist design
- Sense of control and precision
- Relief from digital chaos

**Design Implication:** Clean, confident interface; precise feedback; celebrate with restraint

---

#### **Phase 2: TRUSTED** *(Ongoing Use)*
**The Feeling:** "It understands my work"

Users develop confidence:
- Recognizes intelligent pattern matching
- Trusts the system's decisions
- Feels the app respects their workflow
- Relies on it as a refined tool

**Design Implication:** Consistent, predictable, transparent; respects user intelligence

---

#### **Phase 3: SEAMLESS** *(Long-term Habit)*
**The Feeling:** "Form, perfected"

The ultimate goal - elegant utility:
- Becomes invisible yet essential
- Fades into workflow naturally
- Just part of a refined system
- Missed immediately when absent

**Design Implication:** Minimal presence, maximum impact, quiet sophistication

### Motion & Animation

#### **Animation Principles**

**Purposeful, Not Decorative**
- Every animation should have a reason
- Explain state changes
- Guide attention
- Provide feedback
- Never animate for "wow" factor

**Fast, Not Slow**
- 200-300ms for most transitions
- 150ms for micro-interactions
- 400ms maximum for major changes
- Instant (<100ms) for critical actions

**Natural, Not Mechanical**
- Use ease-in-out curves
- Decelerate into final position
- Match physical intuition
- No bouncing/elastic (too playful)

---

#### **Standard Timings**

```
Micro-interaction:        150ms (hover, button press)
Standard transition:      250ms (navigation, state change)
Large transition:         400ms (modal appear, sheet slide)
Progressive disclosure:   200ms (expand/collapse)
```

**Easing Curves:**
```
Default: ease-in-out
Button press: ease-out (instant feedback)
Menu appear: ease-out (snappy)
Modal dismiss: ease-in (fade away)
```

**SwiftUI Implementation:**
```swift
// Standard animation
.animation(.easeInOut(duration: 0.25), value: someState)

// Spring animation (use sparingly)
.animation(.spring(response: 0.3, dampingFraction: 0.8), value: someState)
```

---

#### **Specific Animations**

**Button Press:**
```
Duration: 150ms
Effect: Scale to 0.95, then back to 1.0
Easing: ease-out
```

**Navigation (Screen transition):**
```
Duration: 300ms
Effect: Slide + fade
Direction: Forward = right to left, Back = left to right
Easing: ease-in-out
```

**Progress Bar Fill:**
```
Duration: Smooth, continuous
Effect: Width increases
Easing: linear (represents real progress)
```

**Success State:**
```
Duration: 400ms
Effect: Checkmark scales in with subtle bounce
Then: Hold for 800ms, fade out over 200ms
Easing: ease-out
```

**Loading Spinner:**
```
Duration: Continuous rotation
Speed: 1 second per rotation
Easing: linear
```

---

#### **Animation Don'ts**

- âŒ No page transitions on every screen (only between major sections)
- âŒ No animations longer than 500ms
- âŒ No parallax effects
- âŒ No 3D flips or excessive transforms
- âŒ No elastic/bouncy animations (too casual)
- âŒ No looping animations unless loading
- âŒ No animating multiple properties at once (keep simple)

---

#### **Reduced Motion**

**Always respect system preference:**

```swift
@Environment(\.accessibilityReduceMotion) var reduceMotion

// Use reduced animations
if reduceMotion {
    withAnimation(nil) {
        // Instant state change
    }
} else {
    withAnimation(.easeInOut(duration: 0.25)) {
        // Smooth animation
    }
}
```

**Reduced Motion Alternatives:**
- Instant state changes (no animation)
- Fade only (no movement)
- Quick snap (100ms or less)
- Never remove functionality, just motion

### Punctuation & Grammar

**Contractions:**

- Never in errors or instructions

- Only in loading states ("Scanning...")
- Never for dramatic pause
- Three dots, no more

- Always use: "Files, folders, and downloads"

- Use numerals: "3 files", not "three files"
- Exception: "one" at start of sentence

**Time:**
- "a few seconds" not specific milliseconds
- "a moment" for short waits
- "this may take a minute" for longer

## Design Principles

Our design decisions flow from these core principles:

### 1. **Clarity Over Creativity**

**What it means:**
Users should never wonder what something does or what will happen when they click. Function is always more important than form.

**In practice:**
- Obvious button labels ("Organize Now" not "Go")
- Clear visual hierarchy (most important thing is most prominent)
- Predictable interactions (buttons look like buttons)
- No mystery meat navigation

**Example:**
âœ… Button labeled "Move 43 Files to Documents/PDFs"  
âŒ Button labeled "Organize" (what will it organize? where?)

---

### 2. **Invisible Until Needed**

**What it means:**
The best interface disappears. Show UI only when it provides value, then get out of the way.

**In practice:**
- Menu bar icon, not dock icon
- No unnecessary notifications
- Empty states that educate, don't decorate
- Keyboard shortcuts for power users

**Example:**
âœ… Menu bar icon shows badge when files need organizing  
âŒ Desktop widget always visible taking up space

---

### 3. **Respect Intelligence**

**What it means:**
Users are smart. Don't over-explain, don't patronize, don't hide complexity out of fear.

**In practice:**
- Show reasoning when helpful ("Matched by Screenshots rule")
- Provide escape hatches (review before auto-organizing)
- Advanced features available but not required
- Trust users to make decisions

**Example:**
âœ… "No rule found" with option to create one  
âŒ "Oops! We're not sure what to do with this file. Don't worry, we won't touch it!"

---

### 4. **Speed is a Feature**

**What it means:**
Every millisecond matters. Fast software feels intelligent, responsive software builds trust.

**In practice:**
- <100ms for UI responses
- Background processing for heavy tasks
- Instant feedback on every action
- Progress indicators for >1 second tasks

**Example:**
âœ… File list appears instantly, thumbnail loads progressively  
âŒ Spinner blocks UI while loading everything

---

### 5. **Native First**

**What it means:**
Embrace macOS patterns. Don't reinvent what Apple already perfected.

**In practice:**
- Use system fonts, colors, controls
- Follow HIG (Human Interface Guidelines)
- Support system features (dark mode, reduced motion, etc.)
- Feel like a Mac app, not a web app

**Example:**
âœ… Standard macOS buttons, menus, shortcuts  
âŒ Custom-styled buttons that don't match any Mac app

---

### 6. **Opinionated Defaults, Flexible Options**

**What it means:**
Make smart decisions for users, but let them override anything.

**In practice:**
- Suggest rules based on file analysis
- Pre-fill forms with intelligent guesses
- Allow customization of everything
- Remember user's choices

**Example:**
âœ… "We suggest organizing screenshots like this [edit]"  
âŒ Empty form with no guidance

---

### 7. **Progressive Disclosure**

**What it means:**
Show complexity gradually. Start simple, reveal depth as needed.

**In practice:**
- Simple onboarding, advanced features discoverable
- Beginner mode vs. power user mode
- Tooltips and help available but not in your face
- Layers of information (overview â†’ details)

**Example:**
âœ… Basic rule builder, then "Advanced options" link  
âŒ All options visible at once, overwhelming

---

### 8. **Error Prevention Over Error Handling**

**What it means:**
Design to prevent mistakes rather than recovering from them.

**In practice:**
- Preview before committing
- Confirm destructive actions
- Disable impossible actions
- Clear what each action does

**Example:**
âœ… "This will move 28 files" preview screen  
âŒ Move first, then show undo notification
